Сгенерируй большой синтетический тестовый датасет для проекта навигации дрона.

Важно: входной поток радиоданных должен быть только в формате NMEA-0183 GPGGA.

Нужно создать следующие файлы:

1. radar_data.nmea
Это основной входной поток.

Требования:
- текстовый файл;
- каждая строка — отдельное валидное сообщение `$GPGGA`;
- использовать только формат `GPGGA`;
- строки должны быть последовательными по времени;
- минимум 50 000 строк;
- поле времени должно меняться последовательно;
- поле высоты должно использоваться как значение радиовысоты;
- высота должна меняться плавно и реалистично;
- контрольная сумма в конце строки должна быть корректной;
- строки должны быть пригодны для чтения стандартным NMEA-парсером.

Пример структуры строки:
$GPGGA,123519.111,5545.1234,N,03736.5678,E,1,08,0.9,145.6,M,46.9,M,,*47

Где:
- время должно быть заполнено;
- широта и долгота должны быть заполнены;
- quality indicator должен быть валидным;
- число спутников должно быть реалистичным;
- HDOP должен быть реалистичным;
- высота должна быть заполнена;
- единицы высоты `M`;
- checksum обязательна.

2. truth.csv
Строгий CSV с заголовком:
timestamp,lat,lon,alt_msl,heading_deg,speed_mps

Требования:
- минимум 50 000 строк;
- timestamp монотонно возрастает;
- lat/lon должны соответствовать траектории, совместимой с NMEA-потоком;
- alt_msl должен меняться плавно;
- heading_deg должен быть согласован с траекторией;
- speed_mps должен быть реалистичным для БПЛА;
- траектория должна включать прямые участки, плавные повороты, набор и снижение высоты.

3. barometer.csv
Строгий CSV с заголовком:
timestamp,baro_alt_m

Требования:
- минимум 50 000 строк;
- timestamp должен совпадать или быть согласуемым с truth.csv;
- baro_alt_m должен быть близок к alt_msl, но содержать реалистичный шум и небольшой дрейф.

4. config.yaml
Создай YAML-файл с полями:
- dem_path
- radar_data_path
- truth_path
- barometer_path
- sample_rate_hz
- correlation
- visualization

Требования:
- пути должны ссылаться на:
  - input/incoming/dem/terrain.tif
  - input/incoming/radar_data.nmea
  - input/incoming/truth.csv
  - input/incoming/barometer.csv
- добавить разумные параметры корреляции;
- добавить разумные параметры визуализации.

5. dem/README.txt
Создай текстовый файл с пояснением, что в папку `dem/` позже нужно положить `GeoTIFF` или `SRTM DEM`.

Дополнительные требования:
- данные должны быть реалистичными для тестирования terrain navigation;
- timestamps должны быть в секундах с дробной частью там, где это применимо;
- данные должны быть согласованными между собой;
- не использовать заглушки;
- не сокращать объем;
- не писать “и так далее”;
- выдать содержимое файлов в готовом виде;
- если вывод слишком длинный, разбивать по частям с четкими заголовками:
  - radar_data.nmea part 1/N
  - truth.csv part 1/N
  - barometer.csv part 1/N

Важно:
- все строки в `radar_data.nmea` должны быть именно `$GPGGA`;
- не использовать другие типы NMEA сообщений;
- checksum должна быть вычислена правильно для каждой строки;
- данные должны быть пригодны для прямой загрузки в проект без изменения кода.
